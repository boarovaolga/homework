Заголовок:   Метод trim удаляет только пробелы, игнорируя табуляцию и другие пробельные символы
Описание:
    Согласно документации, метод trim должен "удалять пробелы в начале" строки.
    В текущей реализации удаляется только символ обычного пробела (U+0020).
    Символы табуляции (\t), перевода строки (\n) и другие
    пробельные символы из Unicode не удаляются, хотя по логике должны.
 Шаги воспроизведения:
    1. Вызвать StringUtils().trim("\ttext")
    2. Вызвать StringUtils().trim("\ntext")
    3. Вызвать StringUtils().trim("\rtext")
Фактический результат:   "\ttext", "\ntext", "\rtext" (строка не изменяется)
Ожидаемый результат:     "text" (все ведущие пробельные символы удалены)
Серьёзность:    Minor (незначительная)
Приоритет:      Low (низкий)
Окружение:      Python 3.x, Windows 11, версия кода 1.0


Заголовок:   Метод contains возвращает True, если вторым аргументом передана пустая строка
Описание:
    Метод contains должен проверять наличие искомого символа/подстроки в строке.
    Однако при передаче пустой строки в качестве symbol метод возвращает True
    для любой непустой строки. Такое поведение не описано в документации и
    противоречит здравому смыслу — пустая строка не является "искомым символом".
Шаги воспроизведения:
    1. Вызвать StringUtils().contains("cmd", "")
Фактический результат:   True
Ожидаемый результат:     False (или явно задокументировано, что пустая строка считается содержащейся)
Серьёзность:    Major (значительная)
Приоритет:      Medium (средний)
Окружение:      Python 3.x,  Windows 11, версия кода 1.0


Заголовок:   Метод contains возвращает True при вызове contains("", "")
Описание:
    Если оба аргумента — пустые строки, метод возвращает True.
    Логически пустая строка не содержит ничего, в том числе пустую строку,
    поэтому ожидаемый результат — False.
Шаги воспроизведения:
    1. Вызвать StringUtils().contains("", "")
Фактический результат:   True
Ожидаемый результат:     False
Серьёзность:    Minor (незначительная)
Приоритет:      Low (низкий)
Окружение:      Python 3.x, Windows 11, версия кода 1.0
